# Contributing to ETFCTA

The Git history is part of ETFCTA's classification memory. Commits MUST use a semantic message that states the affected system and the decision embodied by the change.

Format:

`type(scope): imperative description`

Allowed types: `feat`, `fix`, `validator`, `governance`, `protocol`, `spec`, `data`, `docs`, `test`, `refactor`, `chore`.

Examples:

- `protocol(admission): enforce preserved source versions`
- `validator(sufficiency): derive decision status from active rule`
- `governance: adopt category intelligence charter`

Messages such as `commit`, `update`, `changes`, or `fix` without a meaningful description are prohibited.
